# GitHub 이슈 생성 스킬

이 스킬은 프로젝트의 이슈 템플릿을 기반으로 GitHub 이슈를 생성합니다.

## 사용법

```
이슈 만들어줘: <제목>
작업 등록: <제목>
버그 등록: <제목>
```

## 이슈 타입

### 1. Feature Request (기능 요청)

**트리거 키워드**: 구현, 추가, 기능, 개발, 만들어, 작업

**라벨**: `enhancement`

**본문 템플릿**:

```markdown
## 해결하려는 문제

[기능이 필요한 이유나 배경]

## 제안하는 해결책

[구현할 기능 설명]

## 작업 내용

- [ ] 작업 항목 1
- [ ] 작업 항목 2

## 우선순위

[낮음/중간/높음]
```

### 2. Bug Report (버그 리포트)

**트리거 키워드**: 버그, 수정, 에러, 오류, 안됨, 문제

**라벨**: `bug`, `triage`

**본문 템플릿**:

```markdown
## 버그 설명

[버그에 대한 설명]

## 재현 방법

1. ...
2. ...

## 예상 동작

[정상적으로 동작했다면 어떻게 되었을지]

## 추가 정보

[기타 정보]
```

## ⚠️ 중요: 확인 단계 필수

**이슈를 생성하기 전에 반드시 사용자에게 미리보기를 보여주고 확인을 받아야 합니다.**

### 미리보기 형식

```
📋 이슈 미리보기

**타입**: Feature Request
**제목**: [Feature]: 로그인 페이지 구현
**라벨**: enhancement

---
## 해결하려는 문제
사용자 인증을 위한 로그인 페이지가 필요합니다.

## 제안하는 해결책
로그인 페이지를 구현합니다.

## 작업 내용
- [ ] 로그인 폼 UI 구현
- [ ] 로그인 API 연동
- [ ] 에러 처리 및 유효성 검사

## 우선순위
중간 - 작업 효율 향상
---

이대로 생성할까요? (수정이 필요하면 말씀해주세요)
```

### 확인 후 처리

- **승인**: "응", "네", "생성해줘", "좋아", "ㅇㅇ" → 이슈 생성
- **수정 요청**: 수정 후 다시 미리보기 표시
- **취소**: "취소", "아니" → 생성 취소

## 실행 규칙

1. 사용자 입력에서 키워드를 분석하여 이슈 타입 결정
2. 제목에서 적절한 작업 내용 추론
3. 템플릿에 맞는 본문 자동 생성
4. **미리보기를 보여주고 사용자 확인 대기**
5. 사용자가 승인하면 `gh issue create` 명령어 실행
6. 생성된 이슈 URL 반환

## 명령어

```bash
gh issue create \
  --title "[Feature]: <제목>" \
  --label "enhancement" \
  --body "<본문>"
```

## 다중 이슈 생성

쉼표로 구분하여 여러 이슈를 한 번에 생성할 수 있습니다:

**입력**: `작업 등록: 로그인 페이지, 회원가입 페이지, 마이페이지`

각 이슈마다 미리보기를 보여주고 확인을 받습니다.
