# Git 브랜치 동기화 스킬

리모트에 없는 로컬 브랜치를 삭제하여 리모트와 동기화합니다.

## 사용법

```
브랜치 정리해줘
리모트랑 싱크 맞춰줘
삭제된 브랜치 정리
```

## 실행 규칙

1. `git fetch --prune`으로 리모트 정보 업데이트
2. 리모트에서 삭제된 브랜치 목록 확인
3. **사용자에게 삭제할 브랜치 목록을 보여주고 확인 받기**
4. 승인 시 브랜치 삭제

## ⚠️ 중요: 확인 단계 필수

**브랜치를 삭제하기 전에 반드시 사용자에게 확인을 받아야 합니다.**

### 미리보기 형식

```
🔄 브랜치 동기화

리모트에서 삭제된 브랜치:
- develop
- feature/old-feature

위 브랜치들을 로컬에서 삭제할까요?
```

### 확인 후 처리

- **승인**: "응", "네", "삭제해줘" → 브랜치 삭제
- **취소**: "취소", "아니" → 삭제 취소

## 명령어

```bash
# 리모트 정보 업데이트 및 삭제된 브랜치 정리
git fetch --prune

# 리모트에서 삭제된 로컬 브랜치 찾기
git branch -vv | grep ': gone]' | awk '{print $1}'

# 브랜치 삭제 (병합된 경우)
git branch -d <branch-name>

# 브랜치 강제 삭제 (병합되지 않은 경우)
git branch -D <branch-name>
```

## 주의사항

- 현재 체크아웃된 브랜치는 삭제할 수 없음
- 병합되지 않은 브랜치는 `-D` 옵션으로 강제 삭제
- main/master 브랜치는 삭제하지 않음
